## 二、携宠预约流程（核心主流程）

> 携宠家庭从进入官网到完成预约的完整路径

```mermaid
flowchart TD
    A[用户进入官网] --> B{来源入口}
    B -->|直接访问/搜索| C[首页 Home]
    B -->|社交媒体广告| C
    B -->|邮件/LINE推送| C
    B -->|朋友推荐| C

    C --> D{用户意图判断}
    D -->|想了解宠物服务| E[宠物友好页 Pet-Friendly]
    D -->|想看房型| F[住宿房型页 Stay]
    D -->|直接预约| G[预约页 Booking]

    E --> E1[查看宠物服务清单]
    E1 --> E2[阅读入住规则]
    E2 --> E3[查看宠物温泉/Dog Run]
    E3 --> E4[点击携宠预约CTA]
    E4 --> G

    F --> F1[筛选宠物友好房]
    F1 --> F2[查看房型详情+布局图]
    F2 --> F3[点击预约CTA]
    F3 --> G

    G --> H[Step 1：选择日期与人数]
    H --> H1{日期校验}
    H1 -->|退房≤入住| H_ERR[提示：退房日期需晚于入住]
    H_ERR --> H
    H1 -->|日期有效| I[Step 2：房型偏好]

    I --> I1[选择：宠物友好房/营地小别墅/无偏好]
    I1 --> J{是否携宠？}

    J -->|是| K[Step 3：宠物信息表单]
    K --> K1[填写：宠物类型/数量]
    K1 --> K2[填写：犬种或猫种/体重/年龄]
    K2 --> K3{疫苗证明状态}
    K3 -->|已有| K4[上传或确认疫苗证明]
    K3 -->|未有| K5[提示：入住前需完成接种]
    K4 --> K6[填写特殊需求]
    K5 --> K6
    K6 --> L

    J -->|否| L[Step 4：联系方式]

    L --> L1[填写：姓名/邮箱/电话/国家地区]
    L1 --> L2[选择首选联系渠道]
    L2 --> M[Step 5：规则确认]

    M --> M1[勾选：隐私政策]
    M1 --> M2[勾选：宠物入住规则]
    M2 --> M3[勾选：取消政策]
    M3 --> M4{是否订阅开业/活动通知}
    M4 -->|是-主动勾选| M5[记录订阅偏好]
    M4 -->|否| N
    M5 --> N

    N{提交校验}
    N -->|有未填必填项| N_ERR[高亮缺失字段并提示]
    N_ERR --> M
    N -->|全部通过| O[提交预约]

    O --> O1{提交状态}
    O1 -->|网络失败| O_ERR[提示重试，保留表单内容]
    O_ERR --> O
    O1 -->|重复提交| O_DUP[按钮loading防重复]
    O1 -->|提交成功| P[成功确认页]

    P --> P1[显示确认编号]
    P1 --> P2[展示后续流程说明]
    P2 --> P3[预计回复时间]
    P3 --> P4[社媒/订阅入口]

    O --> Q[系统后台动作]
    Q --> Q1[生成Lead记录]
    Q1 --> Q2[发送用户确认邮件]
    Q2 --> Q3[发送运营通知邮件]
    Q3 --> Q4[入库CRM]
    Q4 --> Q5[记录来源页面/UTM/语言/时间]
```

## 三、普通住宿预约流程

> 不携宠用户的精简预约路径

```mermaid
flowchart TD
    A[用户进入官网] --> B[首页 Home]
    B --> C{用户意图}
    C -->|查看房型| D[住宿房型页 Stay]
    C -->|了解设施| E[温泉与设施页 Facilities]
    C -->|规划行程| F[周边探索页 Explore]
    C -->|直接预约| G[预约页 Booking]

    D --> D1[浏览普通房列表]
    D1 --> D2[查看房型详情]
    D2 --> D3[点击预约CTA]
    D3 --> G

    E --> E1[查看主人温泉/BBQ/咖啡]
    E1 --> E2[点击预约CTA]
    E2 --> G

    F --> F1[查看交通/地图]
    F1 --> F2[浏览行程推荐]
    F2 --> F3[点击预约CTA]
    F3 --> G

    G --> H[Step 1：日期与人数]
    H --> I[Step 2：房型偏好]
    I --> I1[选择：普通房/无偏好]
    I1 --> L[Step 4：联系方式 - 跳过Step3宠物信息]
    L --> M[Step 5：规则确认]
    M --> N[提交]
    N --> O{提交状态}
    O -->|成功| P[确认页 + 确认邮件]
    O -->|失败| ERR[保留内容，提示重试]
```

## 五、活动报名流程

> 宠物社群/KOL 参加主题活动的路径

```mermaid
flowchart TD
    A[用户进入活动页 Activities] --> B[浏览活动日历]
    B --> C{活动筛选}
    C -->|按类型| C1[运动会/摄影/派对/生日/婚礼/季节限定]
    C -->|按时间| C2[本月/下月/全部]
    C1 --> D
    C2 --> D

    D[点击具体活动] --> E[活动详情页]
    E --> E1[查看活动主题/时间/地点]
    E1 --> E2[查看费用/名额/宠物要求]
    E2 --> F{活动状态}

    F -->|已开放报名| G[点击报名按钮]
    F -->|时间待公布| H[点击订阅提醒]
    F -->|已满员| I[加入候补名单]
    F -->|已结束| J[查看活动回顾/照片]

    G --> K[填写报名信息]
    K --> K1[姓名/联系方式]
    K1 --> K2[宠物信息摘要]
    K2 --> K3[特殊需求]
    K3 --> L[确认报名]

    L --> M{报名状态}
    M -->|成功| N[活动报名确认页]
    M -->|失败| ERR[提示重试]

    N --> N1[显示确认编号]
    N1 --> N2[发送活动确认邮件]
    N2 --> N3[后续活动前提醒邮件]

    H --> O[记录邮箱+活动偏好]

    I --> P[记录候补信息]
    P --> P1[有空位时通知候补用户]

    J --> Q[浏览照片画廊]
    Q --> R[分享社媒/下次活动兴趣]
```

## 七、后台Lead跟进流程

> 运营人员从收到Lead到完成确认的闭环

```mermaid
flowchart TD
    A[系统收到新Lead] --> B[CRM自动创建Lead记录]
    B --> C[自动发送用户确认邮件]
    C --> D[运营人员收到通知邮件]
    D --> E[打开CRM查看Lead详情]

    E --> F{Lead类型}
    F -->|携宠预约| G[检查宠物信息完整性]
    F -->|普通预约| H[检查房型偏好]
    F -->|活动报名| I[检查活动状态和名额]
    F -->|候补名单| J[按优先级排列]

    G --> G1{宠物信息完整？}
    G1 -->|完整| K[核实房型匹配]
    G1 -->|不完整| G2[发送补充信息邮件]
    G2 --> G3{用户补充？}
    G3 -->|是| K
    G3 -->|超48h未回复| G4[发送二次提醒]
    G4 --> G5{仍无回复？}
    G5 -->|是| G6[标记为待跟进]
    G5 -->|否| K

    K --> L[确认房型/价格/日期]
    L --> M[联系用户确认]
    M --> M1{用户确认？}
    M1 -->|确认| N[创建Booking Request]
    M1 -->|需修改| O[修改房型/日期]
    O --> M
    M1 -->|取消| P[标记为已取消+原因]

    N --> Q[发送预订确认邮件]
    Q --> R[更新Lead状态为已确认]
    R --> S[入住前T-7天提醒邮件]
    S --> T[入住前T-1天确认邮件]
    T --> U[入住当天-前台接待]
```

## 十、多语言切换流程

> 用户在不同语言间切换的信息保持逻辑

```mermaid
flowchart TD
    A[用户选择语言] --> B{当前页面}
    B --> C[检测URL语言前缀]
    C --> D{目标语言是否有翻译}

    D -->|有| E[加载对应语言内容]
    E --> F[保持当前页面位置]
    F --> G[更新URL前缀 zh/ja/en]
    G --> H[记录语言偏好到Cookie]

    D -->|无| I[提示该页面暂无此语言版本]
    I --> J{用户选择}
    J -->|查看默认语言版本| K[加载日文默认内容]
    J -->|返回原语言| L[保持当前语言]

    H --> M[后续访问自动使用偏好语言]
    M --> N{浏览器语言设置匹配？}
    N -->|匹配| O[自动展示对应语言]
    N -->|不匹配| P[展示日文+语言切换提示]
```
